# TIPS

Projektübergreifende Lösungsmuster und Workarounds, die in diesem Projekt erarbeitet wurden und potenziell in anderen Projekten wiederverwendet werden können. Im Gegensatz zu Tasks (`Projektmanagement/Aufgaben/`) sind diese Dokumente nicht an einen konkreten Vorgang gebunden, sondern als Referenz gedacht.

## Konvention

- Pro Tipp eine eigene Markdown-Datei mit sprechendem Dateinamen.
- Dateiname ASCII (Kleinbuchstaben, Bindestriche, Umlaute ausgeschrieben), Inhalt mit Umlauten.
- Aufbau pro Datei: Problem, Diagnose, Lösung mit Code-Snippet, Verifikation, Quellen, Bezug zum Ursprungs-Task.
- Quellen-Links auf primäre Quellen (Issues, Specs, offizielle Doku), nicht auf Sekundärartikel.
